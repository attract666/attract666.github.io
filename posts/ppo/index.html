大多数介绍LLM视角下的RLHF文章，都会从传统的RL算法开始，通过一系列的公式推导后，才开始讲解PPO算法，使许多同学在还没有遇到PPO算法时就提前放弃了。因此在这篇博客中，我将帮助你轻松了解什么是PPO算法、PPO算法做了什么。

首先我们会简单介绍一下强化学习的基本概念，后面基于这些概念进行讲解；之后我们会从Actor-Critic算法和A2C算法开始，通过这两个算法掌握PPO的本质思想；最后我们会讲解如何从A2C算法到PPO算法。

什么是强化学习

强化学习可以被视为一种学习的框架，它从环境中试错并获得奖励（正向或负向），然后将其作为反馈从而进行学习。那么“它”是谁呢？对于它，我们称为agent，可以是一个模型或是一个可学习的函数。

强化学习的过程包含若干个episode，每个episode包含若干个step。例如：LLM在生成过程中，每一个回答的token，都可以看作是一个step。

第t个step，agent与环境的交互过程包含以下步骤：

  1. agent收到来自环境的状态 $S_{t}$
  2. 基于该状态 $S_{t}$，agent采取动作 $A_{t}$
  3. 环境进入新状态 $S_{t+1}$
  4. 环境会给agent带来一些奖励 $R_{t}$

用LLM中的视角来看，就是LLM进行一次句子生成，那么状态就是当前已经生成的token，动作就是生成一个token，奖励就是最终人类对生成句子的满意程度。

我们希望一个episode中所有奖励之和能够越大越好，因此agent的目标是最大化一个episode中所有奖励之和的期望。而实现这一目标的过程就是agent经历一遍又一遍的强化学习，更新参数。

为什么是所有奖励之和的期望？

因为奖励通常是稀疏的，也就是说奖励模型通常只有在整句话（整个episode）生成完毕后，才会给出最终的评分。在生成过程中，即时奖励往往都是0。例如：prompt是你给我讲个笑话，LLM生成的第一个token会是你，它即不好笑也不难听，如果立刻给它打分，可能只能是0。

因此我们需要将整个过程的奖励加起来。

而之所以是期望的原因是因为，策略具有随机性：LLM本质上是在输出词表中的概率分布，每一次的生成结果并不一致，从而导致从同一个prompt出发，最终得到的episode有无数种可能。因此如果只看一条episode来更新模型并不科学，而使用期望可以保证模型不仅能生成好句子，而且它生成的好句子的平均概率越来越高。

Actor-Critic算法

Actor-Critic算法是PPO算法的基础，包含两个模型——演员和评论家。

  • 演员就是我们最终需要的agent，负责选择动作，在特定状态 $S_{t}$ 下选择动作 $A_{t}$，选择动作受到一个概率分布 $p(A_{t}|S_{t})$ 影响，这个概率分布又被称为策略。
  • 评论家则是预估该动作的价值，也就是从状态 $S_{t}$ 开始，选择动作 $A_{t}$ 后，直到结束为止能够获得的奖励之和的期望 $Q(S_{t}, A_{t})$ ，这种收益又被称为状态动作价值，可以理解为选择这个token后能带来的价值（这并不与上面说到的单个token的奖励冲突）
评论家模型的作用是什么
因为按照标准强化学习的描述，只有整个episode结束后，我们才能获得价值，但我们希望可以提前得知当前动作的价值。因此critic可以看作是一个估计当前step价值的预测模型。

在LLM的视角下，评论家与演员模型都可以使用神经网络来建模

  • 演员通过 $S_{t}$ 输出的是一个概率分布，而我们的LLM采样也是一个概率分布，因此只需要在最后一层添加一个softmax即可
  • 评论家通过 $S_{t}$ 和 $A_{t}$ 输出的是一个标量值，因此我们只需要在最后接上一个线性层即可

那么我们要如何训练这样两个模型呢？ 让我们来看看在一个step中会发生什么

  1. actor收到来自环境的状态 $S_{t}$
  2. actor生成动作 $A_{t}$，然后critic估计状态动作价值 $Q(S_{t}, A_{t})$。actor通过状态价值和自身得到的分布来更新参数，即 $loss = - \log p(A_{t}|S_{t}) Q(S_{t}, A_{t})$
  3. 环境收到 $A_{t}$ 之后给出状态 $S_{t+1}$ ，更新参数后的actor根据新的状态 $S_{t+1}$ 生成新动作 $A_{t+1}$ 。
  4. 环境给出奖励 $R_{t}$ ，critic可以根据环境给出的奖励来更新参数，即 $loss = [Q(S_{t+1}, A_{t+1}) + R_{t} - Q(S_{t}, A_{t})]^{2}$

actor的loss相对比较好理解一点，简单来说就是：我假设有一个可以预测我当前输出token价值的模型，它可以评估我当前输出的token是好还是坏，如果当前输出的token是好，那么我们就鼓励它更容易生成，反之则抑制它。

那么这个预测模型其实就是critic，当前token的价值就是状态动作价值 $Q(S_{t}, A_{t})$：

  • 如果 $Q(S_{t}, A_{t}) > 0$ ，loss要更小，$\log p(A_{t}|S_{t}) Q(S_{t}, A_{t})$ 部分就要更大，注意 $Q(S_{t}, A_{t})$ 是一个标量，因此就要更新参数使得 $p(A_{t}|S_{t})$ 增大。
  • 如果 $Q(S_{t}, A_{t}) < 0$， actor就需要更新参数来减少 $p(A_{t}|S_{t})$ 。

而critic模型实际上是一个预测模型，它的预测能力不一定准，因此我们也要在step中对它训练，它的目标就是估计准当前token。critic在第 $t$ 步给出的 $Q(S_{t}, A_{t})$ 可以被拆解为 $Q(S_{t+1}, A_{t+1}) + r_{t}$ ，其中 $r_t$ 是critic预估的第 $t$ 步的奖励。而现在环境给出了真实的奖励 $R_{t}$ ，$Q(S_{t}, A_{t})$ 与 $Q(S_{t+1}, A_{t+1}) + R_{t}$ 就不相等了，因此critic的loss就是在抹除认知差。

经过上述过程，actor输出越来越准，critic的预测越来越准，最后就可以使用actor来进行推理。

A2C算法

A2C(Advantage Actor-Critic)算法可以看作是Actor-Critic算法的改良。它的想法也很直观，它更改了评估策略的价值选择。当我们在评价一个策略是否优秀时，是去选择最终得到更高的分数的策略作为更优，还是进步幅度更大的策略呢？举个例子来说，就是你与朋友一起准备期末考试，你平时可以考90分，你的朋友只能考60分，但是经过最终的复习，你考了96分，而你的朋友考了95。尽管你的最后得分更高，但是显然你朋友的复习策略更有效，因为你的得分是基于你的平时积累。

而A2C算法则正是使用这种进步幅度来更新actor与critic，这种幅度称之为优势（Advantage）。那么如何衡量这种优势呢？下面给出了A2C算法的一个step：

  1. actor收到环境的状态 $S_{t}$ ，并给出动作 $A_{t}$
  2. 环境收到 $A_{t}$ 之后给出奖励 $R_{t}$ 和新状态 $S_{t+1}$
  3. critic估计状态价值 $V(S_{t})$ 和 $V(S_{t+1})$ 并计算优势 $\text{Adv}(S_{t}, A_{t}) = V(S_{t+1}) + R_{t} - V(S_{t})$
  4. actor通过 $loss = - \log p(A_{t}|S_{t}) \text{Adv}(S_{t}, A_{t})$ 更新参数
  5. critic通过 $loss = [\text{Adv}(S_{t}, A_{t})]^{2}$ 更新参数

其中 $V$ 也被称为状态价值函数，看完上面的过程相比会有很多问题，让我们一一分析

为什么使用状态价值函数来计算优势？而不是状态动作价值

让我们再来看一遍优势是如何表达的，优势是指在状态 $S_{t}$ 下采取动作 $A_{t}$ 带来的收益，要比在状态 $S_{t}$ 下所有可能动作的平均期望收益好多少，即：

$$\text{Adv}(S_{t}, A_{t}) = Q(S_{t}, A_{t}) - V(S_{t})$$

可以发现上面这个优势的定义中就已经存在我们在Actor-Critic算法中说到的状态动作价值，但在Actor-Critic算法中我们使用critic模型来估计 $Q(S_{t}, A_{t})$ ，而现在我们在A2C中又使用critic来估计 $V$ ，这就会导致critic训练方差变大，十分难以收敛。

那么有什么比较好的办法来解决这个问题呢？我们可以使用TD误差，即：

$$Q(S_{t}, A_{t}) \approx R_{t} + V(S_{t+1})$$

注意我们忽略了折扣因子，所以我们就可以得到实际计算的优势：

$$\text{Adv}(S_{t}, A_{t}) = V(S_{t+1}) + R_{t} - V(S_{t})$$

这样一来，我们就可以只使用critic来估计 $V$

LLM视角下的A2C流程

简单来说就是：在LLM的A2C训练中，生成是一整句（episode）完成的，loss是精确到每个token计算的，而参数更新是按batch进行的。

  1. 收集数据（episode级别）——Rollout
  • LLM接收到一个prompt（例如“今天天气怎么样”）
  • actor模型开始自回归生成，逐个token吐出，直到终止符。这样一个完整的句子就是一个episode
  • 在上面的生成token过程中，环境是不会给奖励的，只有当整句话生成完毕，reward model才会对一段话打出一个标量分数
  1. 计算优势（token级别）
  • critic会评估这句话中每一个状态的价值 $V(S_{t})$
  • 因为真实奖励只在最后一步才有，因此我们需要把最后的奖励逆向传递给前面每一个token，通常依靠GAE（广义优势估计）来实现
  • 计算完毕后，这句话中每个token都有专属的两个数据： $\log p(A_{t} | S_{t})$ 和 $\hat{A_{t}}$
  1. 计算loss并更新
  • 我们会使用多个episode，也就是多句话组成的batch来更新参数，避免方差太大
  • 我们会将每个episode中的每个token计算loss累加起来求平均，最后使用平均后的总loss进行一次反向传播更新actor和critic的参数
GAE怎么实现的倒推？

我们实际上只需要算出每个token的优势 $\text{Adv}$ 即可，在上一个问题中我们提到了TD误差，因此我们可以使用TD误差来计算，即：

$$\delta_{t} = \gamma V(S_{t+1}) + R_{t} - V(S_{t})$$

其中 $\gamma$ 是折扣因子，用来控制对未来奖励的看重程度

如果我们只使用 $\delta_{t}$ 作为优势，那么它太短视了，偏差太大（高度依赖critic模型）。因此我们希望早期生成的token也能受到最后奖励的影响，我们需要把未来的 $\delta$ 也计算进来，从而得到：

$$\hat{A_{t}} = \delta_{t} + (\gamma \lambda)\delta_{t+1} + (\gamma \lambda)^{2}\delta_{t+2} + \dots + (\gamma \lambda)^{T-t}\delta_{T}$$

上面的 $\lambda$ 就是为了控制距离当前token越远，累计的误差就越大。此处的 $\lambda$ 非常巧妙：

  • $\lambda = 0$ 时，公式退化为普通的单步优势，偏差大，方差小
  • $\lambda = 1$ 时，公式将高度依赖所有的惊喜，方差极大（因为episode很多），没有偏差
  • 而介于两者之间，则实现了一种折中

但在工程实现中，一般不使用上面的递推式，而是采取下面的办法：

  1. 先正向跑一遍，critic得到t个 $V$
  2. 计算出t个 $\delta$ ，其中第t个 $\delta$ 的 $R_{t}$ 为环境给出的奖励，而其他的 $\delta$ 的 $R_{t}$ 为0
  3. 通过倒推的方式，获得 $\hat{A_{t}}$ 和 $\hat{A_{t-1}} $ 等

从而实现一步步逆向得到前面的优势过程，其中 $\hat{A_{t}}$ 是GAE估计的优势，是一个标量

PPO算法

PPO算法可以看作是一种特殊的A2C算法，它实际上只在A2C上做了一点小的修改，来维持训练的稳定性，想让actor的策略更新幅度不要太大

在PPO中，使用的做法是：

$$loss = - \frac{p(A_{t}|S_{t})}{{p}’(A_{t}|S_{t})} \hat{A_{t}}$$

$p(A_{t}|S_{t})$ 是本次参数更新前的策略，而 ${p}’(A_{t}|S_{t})$ 是上一次参数更新前的策略。上面的式子有一个直观的理解是：把 $\frac{\hat{A_{t}}}{{p}’(A_{t}|S_{t})}$ 看作是一种“学习率”

  • 如果 $\hat{A_{t}} > 0$ 且 ${p}’(A_{t}|S_{t})$ 很大时，整个就会很小，loss还是负数，loss的绝对值很小，优化器就应该增大策略 $p$，但是幅度是小的（即动作 $A_{t}$ 能带来很大的优势，且选择动作 $A_{t}$ 概率也很高，就不必大幅更新参数）
  • 如果 $\hat{A_{t}} < 0$ 且 ${p}’(A_{t}|S_{t})$ 很大时，loss为正，loss的绝对值很小，优化器要往负无穷优化，那么就要减小当前的策略 $p$，幅度也是平稳的
  • 如果 ${p}’(A_{t}|S_{t})$ 很小，那么可能会带来梯度爆炸的风险
为什么使用旧策略会带来梯度爆炸风险，还要使用这种方法呢？

这也正是PPO能大幅降低LLM训练成本的原因：重要性采样

在标准A2C中，actor每与环境交互生成一条数据，更新完参数后，这条数据就需要作废，因为策略变了，旧数据不能用来评估新策略(on-policy)。但在LLM中，生成数据成本高，我们希望旧策略生成的数据，能够被反复利用，对当前策略进行好几次参数更新

当我们用旧策略分布的数据，去优化新策略时，数据分布产生了偏差，而除以 ${p}’$ 就是在数学期望上消除这种分布偏差

针对重要性采样的缺陷，PPO算法还引入了clip操作：

  • 令 $r(A_{t}, S_{t}) = \frac{p(A_{t}|S_{t})}{{p}’(A_{t}|S_{t})}$
  • 当 $\hat{A_{t}} > 0$ 时，若 $r(A_{t}, S_{t}) > 1.2$ 则停止参数更新
  • 当 $\hat{A_{t}} < 0$ 时，若 $r(A_{t}, S_{t}) < 0.8$ 则停止参数更新

因此，整个PPO算法的loss就可以写成：

$$loss = - \min(r(A_{t},S_{t})\hat{A_{t}}, clip(r(A_{t},S_{t}), 0.8, 1.2)\hat{A_{t}})$$

  • 当 $\hat{A_{t}} > 0$ 时，如果 $r(A_{t}, S_{t}) > 1.2$，min操作会取右边的值，loss变为常数，不产生梯度
  • 当 $\hat{A_{t}} < 0$ 时，如果 $r(A_{t}, S_{t}) < 0.8$，min操作会取右边的值，loss变为常数，不产生梯度

上面就是PPO算法的主要改动，但在针对LLM的场景下，PPO的总损失往往会包含3项：

  • 用于更新actor的策略损失： $loss_{policy} = - \min(r(A_{t},S_{t})\hat{A_{t}}, clip(r(A_{t}, S_{t}), 1-\epsilon, 1+\epsilon)\hat{A_{t}})$
    • 往往还会有一项维持模型不要偏离太远的KL散度： $- \beta D_{KL}(p_{actor}(\cdot | S_{t}) || p_{ref}(\cdot | S_{t}))$
    • 当然KL散度并不会作为反向传播的loss，而是提前融合到reward中
    • GAE根据环境给出的真实奖励减去上面的KL散度作为 $R_{t}$ 去计算单步TD误差 $\delta_{t}$ ，并进一步计算 $\hat{A_{t}}$
  • 用于更新critic的价值损失： $loss_{value} = \mathbb{E}[(V(S_{t}) - V_{target})^{2}]$
    • 这里似乎与上面的介绍有点不一样，但只是在工程实现上的不同，本质是等价的。因为我们就是要强化critic在状态t下预测value的能力
    • 而 $V_{target} = \hat{A} + V_{old}(S_{t})$
    • 其中 $\hat{A}$ 是GAE推导出的优势，但是它是个标量值，梯度永远是0，因此实际的工程场景中，使用新的预测值 - 旧的预测值 - 估计的优势
  • 用于鼓励探索的熵损失： $loss_{entropy} = - \sum p(A_{t}|S_{t})\log p(A_{t}|S_{t})$

至此，PPO算法就全部结束了，简单来说就是：根据优势来决定是否强化动作，限制更新幅度，带有clip的on-policy强化学习优化算法
